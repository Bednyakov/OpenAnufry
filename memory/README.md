# 🧠 Система долговременной памяти агента

## Описание

Система долговременной памяти для агента Ануфрий, обеспечивающая:
- **Хранение важной информации** между сессиями
- **Семантический поиск** через OpenAI embeddings
- **Автоматическую классификацию** важности информации
- **Умную очистку** устаревших данных

## Архитектура

### Компоненты

1. **MemoryManager** (`memory/manager.py`)
   - Основной класс для управления памятью
   - Использует SQLite для хранения
   - Интегрируется с OpenAI API для embeddings

2. **Memory Tools** (`tools/memory_tools.py`)
   - Инструменты для LLM агента
   - Обёртки над MemoryManager

### Структура базы данных

#### Таблица `facts` (долговременная память)
- `id` - уникальный идентификатор
- `content` - содержимое факта
- `category` - категория (user_info, solution, preference, skill, general)
- `importance` - важность (1-10)
- `embedding` - векторное представление (JSON)
- `created_at` - дата создания
- `last_accessed` - последний доступ
- `access_count` - количество обращений
- `metadata` - дополнительные данные (JSON)

#### Таблица `conversations` (среднесрочная память)
- `id` - уникальный идентификатор
- `session_id` - ID сессии
- `role` - роль (user/assistant)
- `content` - содержимое сообщения
- `importance` - важность (1-10)
- `created_at` - дата создания
- `metadata` - дополнительные данные (JSON)

#### Таблица `skill_memory` (память о навыках)
- `id` - уникальный идентификатор
- `skill_name` - название навыка
- `usage_context` - контекст использования
- `result` - результат выполнения
- `success` - успешность (boolean)
- `embedding` - векторное представление (JSON)
- `created_at` - дата создания
- `last_used` - последнее использование
- `use_count` - количество использований

## Использование

### Для разработчиков

```python
from memory import MemoryManager

# Инициализация
memory = MemoryManager()

# Сохранение факта
fact_id = await memory.add_fact(
    content="Пользователь предпочитает Python",
    category="user_info"
)

# Поиск релевантной информации
results = await memory.search_facts(
    query="какой язык программирования",
    limit=5
)

# Сохранение диалога
await memory.add_conversation(
    session_id="session-123",
    role="user",
    content="Привет!"
)

# Получение контекста для LLM
context = await memory.build_context_prompt(
    query="текущий запрос",
    session_id="session-123"
)

# Очистка старых данных
result = await memory.cleanup_old_data(days_threshold=30)

# Статистика
summary = await memory.get_memory_summary()
```

### Для LLM агента

Агент имеет доступ к следующим инструментам:

1. **memory_save_fact** - сохранить важный факт
   ```json
   {
     "content": "Пользователь работает в компании X",
     "category": "user_info"
   }
   ```

2. **memory_search** - найти информацию
   ```json
   {
     "query": "где работает пользователь",
     "limit": 5
   }
   ```

3. **memory_get_summary** - получить статистику
   ```json
   {}
   ```

4. **memory_cleanup** - очистить старые данные
   ```json
   {
     "days": 30
   }
   ```

## Классификация важности

Система автоматически оценивает важность информации по шкале 1-10:

- **1-3**: Малозначимая (приветствия, подтверждения)
  - Удаляется через 7 дней при низкой активности
  
- **4-6**: Средняя (настройки, предпочтения)
  - Хранится ~30 дней
  
- **7-10**: Важная (факты о пользователе, решения)
  - Хранится долго, высокий приоритет в поиске

## Семантический поиск

Поиск работает на основе косинусного сходства между embeddings:

1. Запрос преобразуется в вектор через OpenAI API
2. Вычисляется сходство со всеми фактами в БД
3. Возвращаются топ-N наиболее релевантных результатов
4. Обновляется статистика доступа

## Автоочистка

Система автоматически удаляет:
- Факты с важностью ≤3, старше N дней, с низкой активностью
- Диалоги с важностью ≤3, старше N дней

Важная информация (важность ≥7) сохраняется независимо от возраста.

## Оптимизация

### Производительность
- Индексы на важность, категорию, session_id
- Embeddings вычисляются только при сохранении
- Поиск работает только по фактам с достаточной важностью

### Память
- Embeddings хранятся в сжатом JSON формате
- Старые данные автоматически удаляются
- Контекст для LLM ограничен топ-N результатами

## Интеграция с навыками

Система автоматически отслеживает использование навыков:

```python
await memory.add_skill_usage(
    skill_name="mongo-compass",
    usage_context="подключение к MongoDB",
    result="успешно подключено",
    success=True
)

# Получить контекст предыдущих использований
context = await memory.get_skill_context(
    skill_name="mongo-compass",
    query="как подключиться"
)
```

## Конфигурация

Настройки в `config.py`:
- `LLM_API_KEY` - ключ для OpenAI API (embeddings)
- `LLM_BASE_URL` - базовый URL API

База данных создаётся автоматически в `memory/agent_memory.db`.

## Примеры использования

### Сохранение информации о пользователе
```python
await memory.add_fact(
    "Пользователь Артём, работает над проектом OpenAnufry",
    category="user_info"
)
```

### Сохранение успешного решения
```python
await memory.add_fact(
    "Для исправления ошибки OpenAI credentials нужно проверить переменные в .env",
    category="solution"
)
```

### Поиск прошлых решений
```python
results = await memory.search_facts(
    "как исправить ошибку с API ключом",
    limit=3
)
```

## Лучшие практики

1. **Сохраняйте только важное** - не засоряйте память временной информацией
2. **Используйте правильные категории** - это улучшает организацию
3. **Регулярно очищайте** - запускайте cleanup раз в неделю
4. **Проверяйте статистику** - следите за ростом базы данных
5. **Используйте семантический поиск** - он лучше ключевых слов

## Ограничения

- Embeddings требуют API вызовы (стоимость)
- Поиск по большой базе может быть медленным
- SQLite не подходит для высоконагруженных систем
- Максимальный размер embedding: ~1536 измерений
