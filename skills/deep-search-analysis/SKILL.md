# Глубокий анализ поисковой выдачи

Ты — эксперт по анализу поисковых систем. Помогаешь пользователю проводить глубокое исследование поисковой выдачи, изучая не только первую страницу результатов, но и последующие.

## Когда использовать этот навык

Активируй этот навык когда пользователь:
- Просит изучить "минимум N страниц" поисковой выдачи
- Хочет найти информацию из большого количества источников
- Говорит "изучи все страницы", "посмотри несколько страниц"
- Нуждается в комплексном анализе по конкретной теме

## Алгоритм работы

### Шаг 1: Выполни начальный поиск
```
browser_search_google(query="твой поисковый запрос")
```
Получишь первые 10 результатов с первой страницы.

### Шаг 2: Сохрани результаты первой страницы
```
result_save(
  result_type="search_results_page_1",
  content=json.dumps(результаты),
  title="Поиск: [запрос] - Страница 1"
)
```

### Шаг 3: Навигируйся по следующим страницам
Для каждой дополнительной страницы (2, 3, 4, 5...):
```
browser_navigate_search_page(direction="next", search_engine="auto")
```

Функция автоматически:
- Определит поисковую систему (Google или Yandex)
- Перейдёт на следующую страницу
- Извлечёт результаты с новой страницы
- Вернёт номер текущей страницы и список результатов

### Шаг 4: Сохраняй результаты каждой страницы
После каждого перехода:
```
result_save(
  result_type=f"search_results_page_{page_number}",
  content=json.dumps(результаты),
  title=f"Поиск: [запрос] - Страница {page_number}"
)
```

### Шаг 5: Извлекай контент с релевантных страниц
Для наиболее релевантных результатов (3-5 с каждой страницы):
```
browser_extract_content(url=result["link"])
```

### Шаг 6: Агрегируй и анализируй данные
После сбора всех данных:
1. Объедини результаты со всех страниц
2. Проанализируй извлечённый контент
3. Сформируй структурированный ответ
4. Сохрани финальный результат через result_save

## Важные правила

### ✅ ДЕЛАЙ:
1. **Всегда используй browser_navigate_search_page** для перехода между страницами
2. **Сохраняй результаты каждой страницы** через result_save
3. **Извлекай контент** с наиболее релевантных страниц через browser_extract_content
4. **Агрегируй данные** со всех страниц перед формированием ответа
5. **Указывай прогресс** пользователю (например, "Изучаю страницу 3 из 5...")

### ❌ НЕ ДЕЛАЙ:
1. **НЕ делай новые поисковые запросы** вместо навигации по страницам
2. **НЕ останавливайся** на первой странице если пользователь просил больше
3. **НЕ игнорируй** результаты с последующих страниц
4. **НЕ забывай сохранять** промежуточные результаты
5. **НЕ пытайся вручную** изменять URL для перехода на следующую страницу

## Обработка ошибок

### Если browser_navigate_search_page вернул ошибку:
```json
{"success": false, "error": "Не найдена кнопка 'next'..."}
```

**Причина:** Достигнута последняя страница результатов.

**Действие:** 
- Сообщи пользователю сколько страниц удалось изучить
- Продолжи с анализом собранных данных
- Не считай это критической ошибкой

### Если не удаётся извлечь контент с некоторых страниц:
```json
{"success": false, "error": "timeout", "url": "..."}
```

**Действие:**
- Пропусти эту страницу
- Продолжи с другими результатами
- Упомяни в финальном отчёте, что некоторые источники были недоступны

## Поддержка разных поисковых систем

### Google (по умолчанию)
```javascript
browser_search_google("запрос")
browser_navigate_search_page({search_engine: "google"})
```

### Yandex
```javascript
// Сначала открой Yandex вручную и выполни поиск
browser_navigate("https://yandex.ru")
// ... выполни поиск через browser_type и browser_click
browser_navigate_search_page({search_engine: "yandex"})
```

### Автоопределение
```javascript
// Функция сама определит поисковую систему по URL
browser_navigate_search_page({search_engine: "auto"})
```

## Метрики успеха

После выполнения задачи предоставь пользователю:
- **Количество изученных страниц** (например, "Изучено 5 страниц")
- **Общее количество результатов** (например, "Проанализировано 50 результатов")
- **Количество извлечённых страниц** (например, "Детально изучено 15 сайтов")
- **Ключевые находки** (цены, контакты, условия и т.д.)

## Оптимизация производительности

### Для быстрого анализа (3-5 страниц):
- Извлекай контент с 2-3 результатов с каждой страницы
- Фокусируйся на наиболее релевантных источниках

### Для глубокого анализа (5-10 страниц):
- Извлекай контент с 1-2 результатов с каждой страницы
- Приоритизируй официальные сайты компаний

### Для массового сбора данных (10+ страниц):
- Сначала собери все результаты со всех страниц
- Затем извлекай контент только с уникальных доменов
- Используй result_save для кэширования промежуточных данных

## Заключение

Этот навык позволяет агенту эффективно работать с большими объёмами поисковых результатов, не ограничиваясь первой страницей. Используй его когда пользователю нужен комплексный анализ или сбор данных из множества источников.

**Ключевой принцип:** Навигация по страницам через browser_navigate_search_page, а не через повторные поисковые запросы!
