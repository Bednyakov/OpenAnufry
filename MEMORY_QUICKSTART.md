# 🚀 Быстрый старт: Система памяти

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Убедитесь, что в `.env` настроены ключи:
```env
LLM_API_KEY=your-api-key
LLM_BASE_URL=https://openrouter.ai/api/v1
```

3. Запустите агента:
```bash
python main.py
```

## Как это работает

### Автоматическое сохранение
Агент **автоматически** сохраняет все диалоги и оценивает их важность. Вам не нужно ничего делать вручную!

### Использование памяти

#### 1. Сохранение важной информации
Агент может сам сохранять факты:
```
Вы: Меня зовут Артём, я работаю над AI-проектами
Агент: [автоматически сохранит это в категорию user_info]
```

Или вы можете попросить:
```
Вы: Запомни, что я предпочитаю Python для бэкенда
Агент: [использует memory_save_fact]
```

#### 2. Поиск в памяти
```
Вы: Что ты знаешь обо мне?
Агент: [использует memory_search для поиска информации]
```

```
Вы: Как мы решали проблему с API ключами?
Агент: [ищет в памяти прошлые решения]
```

#### 3. Статистика памяти
```
Вы: Покажи статистику памяти
Агент: [использует memory_get_summary]
```

#### 4. Очистка памяти
```
Вы: Очисти старые данные
Агент: [использует memory_cleanup]
```

## Примеры диалогов

### Пример 1: Сохранение предпочтений
```
💬 Вы: Я предпочитаю использовать PostgreSQL для баз данных

🤖 Агент: Понял! Сохраню эту информацию.
[Агент автоматически сохраняет факт в категорию "preference"]
```

```
💬 Вы: Какую БД я предпочитаю?

🤖 Агент: [Ищет в памяти...]
```

### Пример 3: Сохранение решения
```
💬 Вы: Мы исправили ошибку с API ключом, изменив LLM_API_KEY в config.py

🤖 Агент: Отлично! Сохраню это решение на будущее.
[Сохраняет в категорию "solution" с высокой важностью]
```

### Пример 4: Поиск прошлых решений
```

🤖 Агент: [Ищет в памяти...]
Я нашёл в памяти: "Мы исправили ошибку с API ключом, изменив LLM_API_KEY в config.py"
```

## Категории фактов

Агент использует следующие категории:

- **user_info** - информация о пользователе (имя, работа, интересы)
- **solution** - успешные решения проблем
- **preference** - предпочтения и настройки
- **skill** - информация о навыках и их использовании
- **general** - общая информация

## Важность информации

Система автоматически оценивает важность (1-10):

- **1-3**: Временная информация (удаляется быстро)
- **4-6**: Полезная информация (хранится ~месяц)
- **7-10**: Критически важная (хранится долго)

## Где хранятся данные

- База данных: `memory/agent_memory.db`
- Формат: SQLite (можно открыть любым SQLite клиентом)

## Проверка работы

После первого запуска:

1. Скажите агенту что-то о себе
2. Завершите сессию (Ctrl+C)
3. Запустите агента снова
4. Спросите: "Что ты знаешь обо мне?"

Агент должен вспомнить информацию из прошлой сессии!

## Оптимизация

### Семантический поиск
Память использует embeddings, поэтому поиск работает по смыслу, а не по ключевым словам:

```
Сохранено: "Я работаю Python разработчиком"
Запрос: "какой у меня язык программирования"
Результат: ✅ Найдено!
```

### Автоочистка
Малозначимая информация автоматически удаляется через 30 дней. Важная информация сохраняется навсегда.

## Troubleshooting

### Ошибка: "Missing credentials"
Проверьте `.env` файл и убедитесь, что `LLM_API_KEY` установлен.

### Память не работает
1. Проверьте, что создалась папка `memory/`
2. Проверьте, что есть файл `memory/agent_memory.db`
3. Проверьте логи на наличие ошибок

### Медленный поиск
Если база данных большая (>10000 записей), поиск может замедлиться. Запустите очистку:
```
Вы: Очисти данные старше 30 дней
```
